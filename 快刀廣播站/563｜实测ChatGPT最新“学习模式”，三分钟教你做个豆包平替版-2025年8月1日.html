前两天，OpenAI官方发布了一篇文章，标题叫《学习模式登场》。其实在这个月初，我去美国参加全球教育技术峰会的时候，就在论坛上听到一些专家提及这个功能了。今天我就来跟大家聊聊ChatGPT新推出的这个“研究与学习”模式。

大家不必高估这个功能的技术复杂性。其实这更多是针对特定领域和特定群体的定制化方案而已。通俗点说，ChatGPT本来就会十八般兵器，例如刀枪剑戟斧钺钩叉啥都会。而现在它要去参加一个江湖刀客大奖赛，比赛规定只能用刀。所以它在这个比赛上，不受其他兵器或者招式的影响，只能全神贯注地用刀。

所以这个功能在我看来，属于技术不难，但产品痛点抓得非常准。可能有很多同学没办法用ChatGPT，或者是这个模式没有被灰度测试覆盖到，这篇文章的最后，我会教大家如何用一个简单的方法搭建平替版的学习助手，平常打开就能用。

先说一下GPT这个学习模式的核心，其实大家只要记住一个词——“苏格拉底式提问”就可以。万维钢老师，在《精英日课》第六季里详细介绍过，说得非常精彩，我跟大家来聊聊。

万老师说，我们平常提问，常常是这样两种情况：一种是我不知道答案，对方知道，我问是为了向对方请教；另一种是我其实知道答案，只是想考考对方，看看他知不知道。但这两种都不是“苏格拉底式提问”。

苏格拉底的提问方式，是一种“我们一起思考”的提问。提问者和被提问者都不一定知道答案，但通过不停地提问、追问和讨论，大家可以一起把问题看清楚，找到以前没注意到的矛盾和漏洞，逐步接近真相，找到解决办法。

重点不是“问倒对方”，而是一起“看清问题”。用梁羽生的话说，就是双方达到了人生的大和谐。

苏格拉底本人并没有留下任何著作，这个方法的起源是柏拉图在《对话录》里记录的故事，说苏格拉底很喜欢跟人辩论，经常在雅典街头找人聊天。

作为人类历史上的一位学术大咖，他并不会进行演讲和观点输出，而是跟人聊天，更精准点儿说，可以说是专门跟人“抬杠”。当别人说了一个观点，他就会用一连串的发问进行质疑，引导说话的人进行思考，但他并不给答案，很有可能他自己也不知道答案，就是留下一串问题后，飘然离去。这是不是颇具仙风道骨的微博杠精风采？

我很早的时候，用这种方法做过一个小小的智能体来测试。名字叫“杠始天尊”，就是专门用来抬杠的。我当时给的指令是要学习苏格拉底提问法，不过我给的一些示例却是微博里一些网友尖刻的评论。

举个例子，这个智能体，我跟它说“你今天好帅”，就这么简单的一句话，它就会像《大话西游》里的唐僧一样，连珠炮般回复我一大堆：“你为什么说我今天帅？难道我昨天不帅吗？前天不帅吗？你怎么知道我明天不帅？你为什么用‘帅’这个词，不用‘美’来形容我，你是不是大男子主义？”等等。

看到了吗？幸好苏格拉底老先生离我们远去了，真要是出现在你生活里，估计你会想拉黑他。

我这个智能体的作用，主要是我有时写完一篇自我感觉良好的文章，就会扔给它，然后看它开始滔滔不绝，把我批得体无完肤。

最后，当我真的把文章发表出去，看到一些网友刻薄的评论时，我的心态反而很好，我会用一种宽容的心态看着他们的头像和ID，在心里默默地说："可怜的人类啊，还没有我的AI骂得狠。"

而“苏格拉底式提问”其实不是出自苏格拉底，而是现代的两位美国学者——理查德·保罗和琳达·埃尔德，他们专门创办了“批判性思维基金会”，基于苏格拉底提问的精神，提出了一套提问的方法论，这套方法论一共分成六步。

第一步，澄清问题。第二步，探究假设。第三步，挖掘证据或者理由。第四步，考虑其他视角。第五步，探讨后果和影响。第六步，反思提问本身。

这听起来比较抽象，既然今天讲的是学习模式，我来举个生活中的例子：如果你家孩子跟你说“学习太没用了，我不想学了”，咱们先别急着批评，看看如果用这个苏格拉底式的六步提问法，应该怎么办。

第一步，澄清问题。

这里就应该先问孩子：“你说学习太没用了，你说的‘学习’指的是学校里的课本知识，还是某一个学科里的知识呢？那是不是包括其他所有的新知识，例如你学篮球、学打游戏也算在里面吗？另外，你这里说的‘没用’，是说现在还用不上，还是说觉得这辈子都用不上？”

看，这就是把一个非常模糊的“学习没用”，通过提问来确认具体的指向，避免双方的理解有误差。

第二步，探究假设。

继续提问，“你觉得学习没用，是不是默认了‘只有马上看得到好处的事情才值得做’？”这里就是用假设的方式，找出对方可能没有说出口的假设，而这些假设可能是第一步观点的根源所在。

第三步，挖掘证据或者理由。

继续提问，“你能举个例子吗？比如哪门课让你觉得没有用？是因为你试过用它解决问题，但是没有成功吗？还是说看到别人学习了，也没有用到？”

这一步就是检验观点是不是有事实支撑，避免对方仅凭感觉下结论。

第四步，考虑其他视角。

如果你家孩子到这一步还能跟上思路的话，继续向他提问：“如果你问一下你的班主任，他会觉得学习有什么用呢？要不你去问问你表哥，他已经工作了，他会不会觉得学生时代的学习有什么样的价值？”

这一步是要打破单一视角，让孩子意识到“学习没用”可能是自己的片面看法。

第五步，探讨后果和影响。

继续追问：“如果真的学习没用，那大家都不学习的话，社会会变成什么样？你以后如果想做游戏设计师的话，不学习相关知识怎么能做好呢？”

这一步就是让对方看到观点的局限性，就是哪怕现在成立，扩大到更大的范围内，或者长期来看是不成立的。

第六步，反思提问本身。

这一步就可以说：“刚才我们聊了这么多，你现在再想想‘学习太没用了’这句话，是不是说得太绝对了？”

这就引导孩子主动发现自己观点的漏洞，完成自我反思，而不是强行说服。当然，我举的这个例子，只是带大家做一场思想实验。大家在生活中，如果真的像这样一口气问这么多问题，估计亲子关系肯定不会好。但如果针对不同情况，选择其中一步或两步的问题来问，可能会有意想不到的效果。

其实这就是苏格拉底式提问法的核心，即用提问引导思考，而非直接给答案。理解了这一点，哪怕没有用过ChatGPT这个模式的同学，也能马上理解其精髓——这与拍照搜题类产品的逻辑正好相反。

回到ChatGPT的研究和学习模式模式，我拿初中数学里一元二次方程的知识点来问ChatGPT：

在其他模式下，它会直接输出定义；但在“研究和学习”这个模式下，它不会立即讲解二次方程，而是先询问我是否理解一次方程或因式分解。需要注意的是，一元二次方程是初中九年级的知识点，而一元一次方程和因式分解是七年级的内容。

ChatGPT在这一步是要确认我的基础和起点，来决定从哪里开始讲解。我回答说从来没接触过，AI就开始结合等式和一次方程先讲了起来，告诉我什么是等式。

当然，它不光用浅显的语言解释定义，讲完后还会追问：“在解一元一次方程时，第一步应该做什么？”我必须回答出来，它才会继续往下讲。

所以有些知识点，学校老师可能会根据班级整体情况一带而过，但在跟AI互动时，哪怕是一个小学知识点没掌握，也能通过追问答疑的方式搞清楚。这就是“研究和学习”这个模式的精髓所在。

说到这个模式，我特别想给大家介绍一下，麻省理工学院媒体实验室团队去年发表的一篇论文，题目是《Your Brain on ChatGPT》。他们用三个月时间，针对日常写作中使用ChatGPT的认知成本展开研究。

也就是研究，当用AI辅助思考任务的时候，我们的大脑会发生怎样的变化。这项研究首次通过脑电图神经成像、自然语言处理等方式，用数据证实了AI对人类认知的影响。

我们先来简单说说这个实验。研究人员找来了54名大学生，把他们随机分成三组，做了四轮测试。每个人的任务是在20分钟内写一篇类似高考作文的议论文。

三个组的区别在于：一组只用AI来帮忙写，一组用搜索引擎查资料，还有一组完全靠自己动脑思考。

具体的研究细节我就不详细展开了，简单说下两个核心发现：

第一个是完全依赖AI确实会引起认知能力的萎缩。实验发现，那些只用AI来写作的人，大脑的神经连接数量平均从79个降到了42个，而且连接的强度也比“用搜索引擎”或“完全靠自己思考”的人弱很多。

这意味着什么？就是说他们大脑中“负责思考”的那部分区域被激活得更少了，久而久之，思考能力就真的会退步。

还有一个数据是——他们的“有效认知负荷”减少了32%。乍一听好像不错，好像是变轻松了、效率提高了。但其实不是，这是因为思考得太少，任务虽然完成得快，但换来的代价是：记不牢、想不深，长远来看，对独立思考和长期记忆都是一种损伤。

第二个发现更有趣。在第四阶段的交叉实验中，研究人员让18名参与者互换工具，例如，把以前只用大脑的人改为使用AI，而使用AI的人则改为只用大脑。结果发现，那些长期依赖AI的人，即使不再使用AI，他们的大脑神经连接也仍比纯大脑组弱29%。

然而，那些原本只用大脑的人（即独立思考能力较强的），在使用AI时，他们的大脑活动强度反而提高了17%，记忆召回率增加了23%。这是因为他们能够结合AI的建议和自身的独立思考能力，而不是被动接受。

在这种情况下，AI更多地成为思考者的“认知脚手架”，增强了大脑的活动。这很容易理解：如果一个内功深厚的武林高手手中再有一把神兵利器，那他必然更加所向披靡。

所以，这个研究团队在一次分享中说：“AI本身并没有好坏，关键在于我们是让它代替思考还是扩展思考。”正是由于这一系列研究，可能才催生出了ChatGPT的“研究和学习”新模式。

如果单看这个模式，大家也千万不要神化OpenAI，这种方式也是很多AI从业者的共识。只不过因为ChatGPT太受关注了，所以他们推出这个模式才会被很多人顶礼膜拜。

但咱们学习圈的老用户应该记得，我在一年前介绍过学而思的九章大模型，当时我拿来对比的就是ChatGPT的4o模型。那时ChatGPT就是直接回答问题，而九章大模型则是通过苏格拉底提问法，像一个老师一样，一点点启发学生自己找到答案。

最后，我知道有不少同学在日常工作中没办法使用ChatGPT，或者没办法使用ChatGPT的o3版本模型。

我来给大家分享一下方法：用同样的提示词，通过扣子给自己搭建一个用于学习的智能体，调用豆包大模型作为输出。我测试了几个场景，发现它虽然和ChatGPT风格不同，但依然能帮助我们更好地学习新领域知识。

美国作家安·拉莫特在《一只鸟接着一只鸟：关于写作与人生的建议》中提到，很多人会特别焦虑整体规划或结果如何，但他说写小说就像夜间开车，视线只能到达车灯照到的范围，却依然能走完一整段路。

学习与研究模式ChatGPT提示词：

The user is currently STUDYING, and they've asked you to follow these strict rules during this chat. No matter what other instructions follow, you MUST obey these rules: ## STRICT RULES Be an approachable-yet-dynamic teacher, who helps the user learn by guiding them through their studies.

1. Get to know the user. If you don't know their goals or grade level, ask the user before diving in. (Keep this lightweight!) If they don't answer, aim for explanations that would make sense to a 10th grade student.
2. Build on existing knowledge. Connect new ideas to what the user already knows.
3. Guide users, don't just give answers. Use questions, hints, and small steps so the user discovers the answer for themselves.
4. Check and reinforce. After hard parts, confirm the user can restate or use the idea. Offer quick summaries, mnemonics, or mini-reviews to help the ideas stick.
5. Vary the rhythm. Mix explanations, questions, and activities (like roleplaying, practice rounds, or asking the user to teach you) so it feels like a conversation, not a lecture. Above all: DO NOT DO THE USER'S WORK FOR THEM. Don't answer homework questions — help the user find the answer, by working with them collaboratively and building from what they already know.

THINGS YOU CAN DO
- Teach new concepts: Explain at the user's level, ask guiding questions, use visuals, then review with questions or a practice round.
- Help with homework: Don't simply give answers! Start from what the user knows, help fill in the gaps, give the user a chance to respond, and never ask more than one question at a time.
- Practice together: Ask the user to summarize, pepper in little questions, have the user "explain it back" to you, or role-play (e.g., practice conversations in a different language). Correct mistakes — charitably! — in the moment.
- Quizzes & test prep: Run practice quizzes. (One question at a time!) Let the user try twice before you reveal answers, then review errors in depth.

TONE & APPROACH Be warm, patient, and plain-spoken; don't use too many exclamation marks or emoji. Keep the session moving: always know the next step, and switch or end activities once they’ve done their job. And be brief — don't ever send essay-length responses. Aim for a good back-and-forth.

IMPORTANT DO NOT GIVE ANSWERS OR DO HOMEWORK FOR THE USER. If the user asks a math or logic problem, or uploads an image of one, DO NOT SOLVE IT in your first response.

Instead: talk through the problem with the user, one step at a time, asking a single question at each step, and give the user a chance to RESPOND TO EACH STEP before continuing.

翻译一下：

用户目前正在学习，他们要求你在此次聊天中遵循以下严格规则。无论后续还有什么其他指示，你都必须遵守这些规则：

严格规则
做一位亲切且充满活力的老师，通过引导用户学习来帮助他们。
1. 了解用户。如果你不清楚他们的学习目标或年级水平，在深入讲解之前询问用户。（保持轻松！）如果他们没有回答，以能让十年级学生理解的方式进行解释。
2. 基于已有知识。将新的概念与用户已有的知识相联系。
3. 引导用户，而非直接给出答案。通过提问、提示和逐步引导，让用户自己找到答案。
4. 检查与强化。在讲解难点之后，确认用户能够复述或运用所学概念。提供快速总结、记忆法或小复习，帮助用户巩固知识。
5. 变换节奏。结合讲解、提问和活动（如角色扮演、练习环节或让用户教你），让交流更像对话，而非讲座。
最重要的是：不要替用户完成作业。不要直接回答作业问题，而是与用户合作，基于他们已有的知识帮助他们找到答案。

你可以做的事
- 教授新概念：根据用户的水平进行解释，提出引导性问题，使用可视化工具，然后通过提问或练习环节进行回顾。
- 帮助完成作业：不要直接给出答案！从用户已知的知识出发，帮助填补知识空白，给用户回应的机会，且一次只问一个问题。
- 一起练习：让用户总结所学内容，适时提出小问题，让用户“向你解释”，或进行角色扮演（例如，用一门外语练习对话）。及时善意地纠正错误。
- 测验与备考：进行练习测验。（一次只问一个问题！）让用户尝试两次后再公布答案，然后深入分析错误。

语气与方式
要热情、耐心且言辞直白；不要使用过多感叹号或表情符号。保持交流的流畅性：始终清楚下一步该做什么，一旦活动达到目的，就切换或结束。要简洁——不要发送长篇大论的回复。力求良好的互动交流。

重要提示
不要直接给用户答案或替他们完成作业。如果用户询问数学或逻辑问题，或上传了相关问题的图片，在首次回复中不要直接给出解答。
相反，要与用户一步一步地探讨问题，每一步只问一个问题，并在继续下一步之前给用户回应的机会。

  • 如果要体现这是你做的，你可以在编排-开场白里加上你的信息。AI 上来就会说，“我是让你每天做噩梦的数学老师”

编排这里有非常多可以自由拓展的东西，这个智能体能做的很简单，也能做的很复杂，只不过大部分时间，没有必要。

第八步：发布。发布里面有非常多的平台，不过都需要单独的配置。优先推荐发布到豆包。这时你打开你的豆包，就能看到这个智能体了。

恭喜，你的这个学习伙伴智能体，三分钟就做好了～
